## 1. 프로젝트 배경

### 문제 상황

AI(예: Claude, ChatGPT)가 다이어그램을 생성할 때 Mermaid 문법으로 코드를 출력하는 경우가 많다. 하지만 이 코드를 실제 다이어그램 이미지로 변환하려면:

1. **별도 도구 필요**: Mermaid Live Editor 같은 외부 사이트를 방문하거나 로컬 도구 설치 필요
2. **복사-붙여넣기 과정**: AI가 생성한 코드를 복사해서 다른 도구에 붙여넣는 번거로움
3. **실시간 수정 불편**: 코드를 수정할 때마다 수동으로 새로고침하거나 재변환 필요
4. **통합 부족**: AI 워크플로우와 다이어그램 렌더링이 분리되어 있음

### 도메인 설명

**Mermaid**는 텍스트 기반 다이어그램 생성 도구로, 간단한 문법으로 다양한 종류의 다이어그램(플로우차트, 시퀀스 다이어그램, 클래스 다이어그램, ER 다이어그램, Gantt 차트 등)을 작성할 수 있다.

예시:
```mermaid
graph LR
    A[시작] --> B[처리]
    B --> C[종료]
```

## 2. 문제 정의

**핵심 문제**: AI가 생성한 Mermaid 코드를 즉시 시각화하고, 실시간으로 수정 결과를 확인할 수 있는 통합 솔루션이 없다.

**세부 문제**:
1. AI 출력 → 이미지 변환 과정이 수동적이고 번거로움
2. 코드 수정 시 실시간 프리뷰가 제공되지 않음
3. 다양한 환경(웹, API, MCP)에서 일관되게 사용하기 어려움
4. 빠른 이미지 export 기능 부족

---

## 3. 해결 방법

### 핵심 솔루션

**Mermaid Saiko**는 Mermaid 코드를 받아 실시간으로 다이어그램을 렌더링하고, 이미지로 export할 수 있는 서비스를 제공한다.

### 주요 기능

1. **실시간 렌더링**:
   - 웹 에디터에서 코드 수정 시 자동으로 다이어그램 업데이트 (live preview)
   - 디바운싱으로 부드러운 사용자 경험 제공

2. **다중 인터페이스**:
   - **웹 애플리케이션**: 브라우저에서 직접 사용 가능
   - **REST API**: 다른 애플리케이션에서 호출 가능
   - **(향후) MCP 서버**: AI 도구와 직접 통합 가능

3. **이미지 Export**:
   - PNG, SVG 등 다양한 포맷 지원
   - 웹에서 즉시 다운로드 가능
   - API를 통한 프로그래밍 방식 export

4. **완전한 Mermaid 지원**:
   - 모든 Mermaid 다이어그램 타입 지원
   - 최신 Mermaid 버전 사용

### 기술적 접근

- **DDD 기반 설계**: 도메인 로직을 독립적인 모듈로 구현하여, 다양한 인터페이스(REST API, MCP)로 확장 용이
- **Docker 기반 배포**: 로컬 환경에서 쉽게 실행 가능하며, 클라우드 배포도 간편
- **TypeScript + NestJS**: 타입 안전성과 확장 가능한 아키텍처

---

## 4. 기대 효과

### 사용자 관점

1. **워크플로우 단축**: AI 출력 → 이미지 변환 과정이 즉각적
2. **생산성 향상**: 실시간 프리뷰로 빠른 피드백 루프
3. **유연한 사용**: 웹, API, (향후) MCP 등 다양한 방식으로 접근 가능
4. **접근성**: 별도 계정/인증 없이 누구나 사용 가능

### 기술적 관점

1. **확장 가능한 아키텍처**: DDD 설계로 새로운 인터페이스 추가 용이
2. **재사용 가능한 모듈**: 렌더링 로직을 독립적으로 관리
3. **배포 편의성**: Docker 컨테이너로 어디서나 실행 가능

## 5. 사용자 주체

### 주요 사용자 (Primary)

- **본인**: AI 워크플로우에서 Mermaid 다이어그램을 즉시 시각화

### 향후 확장 사용자 (Future)

- **개발자**: API를 통해 자신의 애플리케이션에 Mermaid 렌더링 기능 통합
- **AI 도구 사용자**: MCP 서버를 통해 AI 에이전트가 직접 다이어그램 생성
- **일반 사용자**: 웹 인터페이스를 통해 간단히 Mermaid 다이어그램 작성
